# Mainframe Assessment Readiness Checklist

**System**: [System Name]
**Date**: [YYYY-MM-DD]
**Assessor**: [Name]

## Purpose

Ensure comprehensive legacy system assessment before modernization planning.

## System Inventory

- [ ] Hardware/software inventory completed
- [ ] All programming languages identified and versioned
- [ ] Source code repository established or cataloged
- [ ] Database schemas documented
- [ ] File layouts (VSAM, sequential, indexed) documented
- [ ] Integration points mapped (external systems, APIs, files)
- [ ] Batch job inventory completed (JCL, schedules)
- [ ] Screen/report inventory completed
- [ ] Middleware components documented (CICS, IMS, MQ)

## Code Analysis

- [ ] Lines of code counted by language
- [ ] Cyclomatic complexity calculated
- [ ] Code quality metrics collected
- [ ] Dead code identified
- [ ] Code duplication analyzed
- [ ] Call graphs generated
- [ ] Dependency analysis completed
- [ ] Technical debt cataloged

## Business Analysis

- [ ] Business capability map created
- [ ] Critical business rules extracted
- [ ] Business processes documented
- [ ] SME interviews conducted
- [ ] Existing documentation reviewed
- [ ] Regulatory/compliance requirements identified
- [ ] User roles and permissions documented

## Data Analysis

- [ ] Data models documented
- [ ] Data volumes quantified
- [ ] Data quality issues identified
- [ ] Data growth rates calculated
- [ ] Data retention policies documented
- [ ] Data privacy/security requirements identified

## Integration Analysis

- [ ] All external system integrations cataloged
- [ ] Integration protocols documented
- [ ] Integration frequency/volume documented
- [ ] Integration criticality assessed
- [ ] API contracts documented (if applicable)

## Performance & NFRs

- [ ] Performance baselines established (TPS, response time)
- [ ] Availability requirements documented (uptime SLA)
- [ ] Security requirements identified
- [ ] Compliance requirements documented (SOX, GDPR, HIPAA)
- [ ] Disaster recovery requirements documented (RTO, RPO)

## Modernization Readiness

- [ ] Complexity score calculated (0-100)
- [ ] Risk assessment completed
- [ ] Modernization approach recommended (rehost/replatform/refactor/replace)
- [ ] Effort estimate provided (person-months)
- [ ] Timeline estimate provided
- [ ] Budget estimate provided
- [ ] Resource requirements identified
- [ ] Success factors documented

## Sign-Off

- [ ] Assessment complete
- [ ] Reviewed by stakeholders
- [ ] Approved for next phase
